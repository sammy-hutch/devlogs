## Initiate rich markdown documentation
**Project:**

[homm3bg-rules](https://github.com/sammy-hutch/homm3bg-rules)


**Details:** 

Heroes of Might and Magic III: The Board Game was released on gamefound in 2 waves, with each wave containing multiple expansions. Each expansion came with a hefty booklet of new scenarios and campaigns, but crucially also new rules/components related to the expansion and updates/clarifications to existing rules/components were included in these booklets. This means trying to find a particular rule or explanation of a particular component or mechanic could involve leafing through about 12 booklets if you didn't know where to find what you were looking for, and you may come across an incorrect ruling and not find the updated version.

This prompted me to create my own updated & condensed "rulebook" and reference for the game, which is what the "homm3bg-rules" repo is. 

This is a good chance for me to practice richer markdown documentation than I've been used to before, so with the first commits I've been trying out in-line images, hyperlinks & alt-text on images, and light css/html styling to create containers, side-by-side image and text blocks, etc.
